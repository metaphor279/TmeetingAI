# 腾讯会议总结助手

基于 tmeet CLI 拉取的**真实**会议数据生成可视化总结卡片。数字必须来自真实统计，严禁编造。
所有中间产物写入系统临时目录（如 `/tmp/tmsummary/`），最终 HTML 写入当前工作目录并用 present_files 展示。

## 总流程（严格按序）

### 第 1 步：检验腾讯会议连接与授权

```bash
tmeet auth status --format json 2>&1
```

- 若输出 `Logged in` 且 AccessToken `valid` → 已连接授权，**直接跳到第 3 步**。
- 若命令 **not found**（未安装 CLI）→ 进入第 2 步的「安装」分支。
- 若已安装但未登录（`user config is empty` 或未 Logged in）→ 进入第 2 步的「授权」分支。

### 第 2 步：引导连接（仅在未连接时）

先明确告知用户「需要先连接腾讯会议才能读取会议数据」，**征得同意后**再执行。

**安装分支**（CLI 未安装）——参考 https://github.com/TencentCloud/tencentmeeting-cli ，执行：
```bash
npm install -g @tencentcloud/tmeet@latest
```
安装完成后继续授权分支。

**授权分支**（已安装未登录）——登录必须后台运行以便立即捕获授权 URL：
```bash
tmeet auth login 2>&1 &
```
从输出中提取授权 URL，**必须完整展示给用户**，并明确提示：请在浏览器中打开该链接完成授权。
等待用户完成后，再次 `tmeet auth status` 确认已登录。

> 若环境无默认浏览器：直接告知用户在终端手动执行 `tmeet auth login`，拿到 URL 后在浏览器打开授权。

### 第 3 步：让用户选择总结类型

授权确认后，向用户提供选项（用 AskUserQuestion）：

1. **季度热力图** — 看你近一个季度的开会密度日历（轻量，秒出）
2. **会议人格** — 称号 + 雷达图 + 一句话评价 + 六维数据（默认轻量版，简洁呈现，秒出、覆盖全部会议）

也允许"两个都要"。

> 会议人格默认用**轻量版**（只读会议元数据，无需录制转写）。**优先用轻量版**——因为读取录制/转写在会议或内容量大时极易超时失败，且只能覆盖有录制的少数会议。仅当用户明确想要"基于发言内容"的深度画像（口头禅/提问/控场等）时，才走深度版。

### 第 4 步：按选择执行流水线

完整命令见 [references/pipeline.md](references/pipeline.md)。**执行前务必先读该文件**，其中包含日期窗口处理、分页去重、抓取步骤与六维口径。

概要：
- **季度热力图**：`collect_meetings.sh` → `heatmap_stats.py` → `render_heatmap.py`，产出 `季度会议热力图.html`。
- **会议人格（轻量版·默认）**：`collect_meetings.sh` →（逐会议 `meeting get` 判主持人 + 逐会议 `record list` 判有无录制）→ `persona_light_stats.py` → `render_persona_light.py`，产出 `会议人格卡-轻量版.html`。六维全部来自元数据、不读录制内容：参与/投入/敏捷/规划/主动/沉淀。呈现为**暗色科技 HUD 风**人格卡：深空黑蓝底 + 网格纹 + 四角 HUD 描边 + 青蓝霓虹发光雷达图，称号纯渐变文字（无高亮圈）+ 中心综合战力 + 左侧霓虹竖线的一句话评价 + 六维数据格（顶部发光细线+编号），**不含标签墙/评级角标等游戏化元素**。需要「本人 open_id」（`tmeet auth status` 里的 OpenId）。**完整评选逻辑（六维口径/评分锚点/段位/称号形态规则/一句话模板）见 [references/persona_logic.md](references/persona_logic.md)。**
- **会议人格（深度版·可选）**：`collect_meetings.sh` →（逐会议 `record list` → 逐文件 `transcript-get`）→ `persona_stats.py` → `render_persona.py`，产出 `会议人格卡.html`。仅在用户明确要基于发言内容时使用；需要「本人 user_id」。

### 第 5 步：展示

用 present_files 打开生成的 HTML，并用一段话口播关键结论（最忙一天、称号等）。

## 关键约束

- **数据真实**：所有数字来自 tmeet 统计，任何指标都不得编造或估算。
- **90 天上限**：`meeting list-ended` 时间区间 >90 天会报错，起点设为今天前 89 天。
- **去重**：会议按 `meeting_id + sub_meeting_id` 去重；同一会议多份录制取段落最多的一份。
- **优先轻量、慎用录制**：会议人格默认走轻量版；深度版依赖录制转写，量大时易超时，且 `meeting get` 对部分会议返回错误（无权限/已回收），脚本已容错跳过。
- **仅本人**：人格统计只计本人（open_id / user_id）相关的数据。
- **视觉规范**：热力图为浅蓝→白渐变、扁平无阴影、淡蓝细边框的简洁风格（主色 `#2B5FF6`）；会议人格（轻量版）为**暗色科技 HUD 风**（深空黑蓝底、青蓝霓虹 `#5ad8ff` 描边与发光、四角 HUD 角标、网格纹背景），两者视觉体系不同、各自固化，一般无需改动。
- 中间文件放临时目录，产物放当前工作目录。
