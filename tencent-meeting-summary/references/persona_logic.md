# 会议人格评选逻辑（轻量版 · 完整规则 v2）

> 全流程确定性、可复现、无随机。输入为 tmeet 真实会议元数据，不读取任何录制内容。
> 输出三件套：**六维评分 → 人格称号 + 一句话评价 → 数据呈现**。呈现风格为暗色科技 HUD 卡（深空黑蓝底 + 青蓝霓虹发光 + 四角描边 + 网格纹），不含标签墙/评级角标/五角星等游戏化元素。

---

## 一、六维定义与原始口径

| 维度 | 展示名 | 原始指标 | 采集方式（tmeet） | 方向 |
|---|---|---|---|---|
| 参与 | 参与 | 近90天参会总场次 | `meeting list-ended` 去重（meeting_id+sub_meeting_id） | 越大越高 |
| 投入 | 投入 | 会议总时长（小时） | 各场 (end-start) 累加，单场封顶 600 分钟 | 越大越高 |
| 敏捷 | 敏捷 | 平均单场时长（分钟） | 总时长 / 场次 | **越小越高（反向）** |
| 规划 | 规划 | 预约会占比（%） | 主题非"快速会议"的场次 / 总场次 | 越大越高 |
| 主动 | 主动 | 本人主持会议占比（%） | 逐会议 `meeting get`，`current_hosts` 含本人 open_id / 有详情会议数 | 越大越高 |
| 沉淀 | 沉淀 | 有录制会议占比（%） | 逐会议 `record list`，`total_count>0` / 总会议数（**只判有无，不读内容**） | 越大越高 |

> 说明：主动、沉淀用「占比」而非绝对数，避免"会多"的人天然占便宜；敏捷特意做成反向，鼓励高效短会。

## 二、评分：锚点分段线性插值 → 0–100

每维用一张 `(原始值, 分值)` 锚点表，落在区间内线性插值。锚点是按真实分布校准的：**普通用户约落 50 分，重度用户 85+**。

```
参与 : (0,0)(20,35)(50,55)(90,72)(130,85)(200,100)
投入 : (0,0)(20,35)(50,55)(100,75)(150,88)(250,100)
敏捷 : (15,100)(30,90)(45,78)(60,60)(75,45)(90,30)(120,10)   ← 反向
规划 : (0,0)(20,30)(40,50)(60,68)(75,82)(90,95)(100,100)
主动 : (0,0)(10,35)(20,55)(35,72)(50,85)(70,100)
沉淀 : (0,0)(5,25)(15,45)(30,65)(50,80)(70,90)(100,100)
```

**综合战力** = 六维评分算术平均（展示在雷达图圆心）。

## 四、人格称号（整体分布驱动，唯一，追求传播性）

称号采用**两段式口语结构**（对齐"越聊越上头的点子王"的语感）：**口语状态前缀 + 有梗身份名**。
分两步：**判分布形态（定前缀）→ 取最高维（定身份）**。全能型例外，用独立整句称号。
所有措辞均为正向/中性，不出现负面描述。

### 步骤 1：口语状态前缀（互斥，按优先级从上到下取第一个命中，全部正向）

设六维中 `max` 最高分、`min` 最低分、`avg` 均值、`s1/s2` 为第一/第二高分。

| 优先级 | 形态 | 判定条件 | 前缀 | 说明 |
|---|---|---|---|---|
| 1 | 全能型 | 最低分 ≥ 80 | —（独立称号）| 六维皆强，直接叫「**会议King**」|
| 2 | 均衡型 | max − min ≤ 15 | Everything都拿手的 | 各维接近、全面均衡 |
| 3 | 专精型 | s1 ≥ 85 且 s1 − s2 ≥ 12 | Very专业的 | 单维尖子、明显领先 |
| 4 | 双核型 | s1 ≥ 75 且 s2 ≥ 75 | 越开越起劲的 | 两个维度双高 |
| 5 | 资深型 | avg ≥ 68 | 游刃有余的 | 整体偏上、成熟老练 |
| 6 | 成长型 | 其余全部情况 | 越来越有节奏的 | 整体在稳步积累中 |

### 步骤 2：身份名（= 最高分维度的有梗身份，全部正向/中性）

| 最高维 | 身份名 |
|---|---|
| 参与 | 会议钉子户 |
| 投入 | 会议收割机 |
| 敏捷 | 会议速通选手 |
| 规划 | 会议操盘手 |
| 主动 | 会议节奏大师 |
| 沉淀 | 会议收藏家 |

### 拼接
`称号 = 前缀 + 身份名`（全能型除外）。
组合空间：5 前缀 × 6 身份 + 1 独立称号 = **31 种**。

**示例（本人真实数据）**：六维 = 参与85 / 投入85 / 敏捷56 / 规划76 / 主动50 / 沉淀88。
→ 最低56、max−min=38（非均衡）；s1=88、s2=85，差3（非专精）；前两高均≥75（双核型命中）。
→ 前缀「越开越起劲的」+ 最高维沉淀→身份「会议收藏家」= **越开越起劲的会议收藏家**。

## 五、一句话评价（模板生成，正向措辞）

固定结构：**最强两维（表现突出）+ 最低一维（温和点出、给正向建议，不用"短板/偏低"等负面词）**。

建议话术库（按最低维取，均为鼓励式）：
- 参与：多参与一些讨论，存在感会更强
- 投入：时间安排张弛有度，挺好
- 敏捷：会议节奏可以再明快一点
- 规划：多提前预约，节奏会更从容
- 主动：不妨多主动发起几场，带带节奏
- 沉淀：重要的会开个录制，方便日后回看

模板：
> 你在「{最高维}」「{次高维}」上表现突出，一个季度 {总时长} 小时投入在 {总场次} 场会议里；要说还能再进一步的，「{最低维}」值得点一下——{对应建议}。

## 六、降级与边界

- 拿不到本人 open_id → **主动**维按 0 计（其余五维照常）。
- 不跑 `record list` → **沉淀**维按 0 计。
- `meeting get` 对部分会议返回错误（无权限/已回收）→ 容错跳过，主动占比按"成功取到详情的会议"为分母。
- 所有比例分母口径已在脚本固化，保证同一份数据多次运行结果一致。
